Phase 7.10: Catalog Modernization - JSON catalog, automated validation, enhanced validator, documentation

This commit implements a comprehensive model catalog modernization system to fix Anthropic
smart chunking 404 error and prevent future provider model issues.

## Problem Solved
⚠️ Original Issue: Anthropic smart chunking failed with 404 error for `claude-3-5-sonnet-20241022`
⚠️ Root Cause: Hardcoded model catalog difficult to maintain, no deprecation detection
⚠️ Impact: Users couldn't use smart chunking with Anthropic models

## Solution Implemented

Externalized model catalog to community-editable JSON with automated validation:

**Architecture:**
- catalog/models.json - JSON source of truth for model metadata
- catalog_manager.py - Loads and caches catalog with deprecation tracking
- Enhanced validator - Fetches models from Anthropic API for fresh validation
- CI/CD workflow - Validates all PR changes before merge
- Deprecation tracking - Built-in lifecycle management
- Comprehensive documentation - Community guidelines and developer journal

## Technical Details

**Catalog Features:**
- JSON format for easy editing and version control
- JSON schema validation enforces data quality
- Dated model IDs only (YYYYMMDD pattern)
- Deprecation fields: deprecated, deprecated_date, replacement_model
- Backward-compatible API via catalog_manager
- Cached loading for performance

**Enhanced Validator:**
- Fetches fresh model list from client.models.list() API
- Detects deprecated models (in catalog but not in API)
- Fallback for older SDK versions
- Performance tracking (validation time in ms)

**CI/CD Workflow:**
- Runs automatically on PRs modifying catalog/
- Validates JSON syntax and schema compliance
- Checks required fields, pricing, context windows
- Enforces dated model IDs

**Documentation:**
- AGENTS.md updated with Phase 7.10 complete status
- README.md includes Model Catalog section
- docs/developer-journal.md tracks implementation journey
- docs/CATALOG_MANAGEMENT.md provides community contribution guide
